## Question

如何以最小改动让 `#bear-overlay` 稳定显示在 Lightweight Charts 内部 Canvas 之上，同时保持 `pointer-events: none`、现有 WMA/熊市算法、图表手势、缩放平移、十字光标和横竖屏重绘不变？需提交实现、非空覆盖层绘制断言、层叠契约回归测试，并通过 Android 工程的相关静态构建。

## Resolution

已在独立 worktree 分支 `codex/android-wma-overlay` 完成最小层叠修复与回归保护，提交为 `ee6e1a496c64f0ff232f1879b7c81215ebfd39d1`（`Fix Android WMA overlay layering`）。

### 范围确认

- 按用户在[确定 WMA 白线的产品与验收边界](0001-确定WMA白线的产品与验收边界.md)中的决定，同一个 `#bear-overlay` 的白色 WMA 与绿色熊市填充一起恢复、一起保护。
- 将覆盖 Canvas 的 `z-index` 从 computed `auto` 修为显式 `3`，严格高于锁定的 Lightweight Charts 4.2.3 内部最高层 `2`；保留 `pointer-events: none`。
- 未修改 WMA 数学定义、`close < WMA` 判定、默认周期、白线颜色/宽度、绿色填充值、手势、缩放平移、十字光标、重绘调度或第三方库。

### 改动文件

- `android-app/app/src/main/assets/chart/chart.css`：为 `#bear-overlay` 增加 `z-index: 3`；旁注说明该非显然数值对应 Lightweight Charts 4.2.3 的内部 Canvas 层 `1/2`。
- `android-app/app/src/main/assets/chart/renderer.js`：初始化后读取真实 computed style，比较覆盖层与 `#price-chart` 内部 Canvas 的层级，并验证指针穿透；契约失效时用稳定错误 fail fast。
- `android-app/app/src/test/js/renderer.test.js`：非空 `bearOverlay` 夹具实际记录绿色 `fill` 与 `#ffffff` / 2px `stroke`；读取随 APK 发布的真实 `chart.css` 并经过同一层叠校验器与内部 `1/2` 层比较；另证明 `auto`、`<=2` 和非 `none` 指针行为会在初始化时被拒绝。旧缺陷的 `z-index:auto` 因而会使测试失败，而非仅断言一个脱离运行时的常量。

### 风险与后续注意事项

- `z-index: 3` 和最低内部最高层 `2` 都绑定当前锁定的 Lightweight Charts 4.2.3。运行时校验会读取所有内部 Canvas 的 computed style；未来第三方层级升高时会明确失败，届时应重新取证并同步调整契约，不能静默遮挡。
- Node 回归读取真实发布 CSS，并以可控 computed-style/内部 Canvas 模型验证契约；它不能替代 Android WebView 的最终合成像素证据。后续真机票据仍须用 CDP 确认覆盖层 computed `z-index=3`、内部最高 `2`、`pointer-events:none`，并用截图证明白线与绿色填充实际位于 K 线之上。
- Step 2 集成应合入上述单一提交后重跑全量 JS、JVM、lint 和 debug assemble；不要在合并时重写算法、样式或升级第三方库。
