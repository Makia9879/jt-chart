# 现有图表行为与算法契约

## 用途与真值来源

本文档固化 `jt-regime-oscillator.html` 在 Android MVP 迁移时需要保持的可见行为和计算语义。如果文档与当前实现冲突，回归测试以本文档所指向的固定夹具与预期值为准；算法不得在 Android 网络层重写。

真值来源：

- 图表页：`jt-regime-oscillator.html`
- 现有对照实现：`jt_shared.py`
- 现有回归：`tests/test_jt_regime.py`

通知模块、后台 Worker、介质信息和相关 API 均不属于本契约。

## 输入数据契约

- WebView 接收按开盘时间升序排列的 K 线；每根至少有 `time/open/high/low/close`。
- `time` 是 UTC Unix 秒整数。当原始数据为毫秒时使用 `floor(ms / 1000)`。
- OHLC 进入 JavaScript 后均为 IEEE-754 binary64 `Number`；交易量不参与图表或算法。
- 价格轴精度由原始 OHLC 字符串的最多小数位推断，范围 `0..12`；`minMove = 10^-precision`。这是显示规则，不先舍入计算输入。
- 页面本身包含当前未闭合 K 线。Android MVP 如果保持当前图表语义，展示和计算都使用行情源返回的整组 K 线；`closed_candles(..., settle=10s)` 是旧后台通知语义，不应带入无通知 App。

## 设置、默认值与校验

| 设置 | 默认 | 可选/读取规则 |
| --- | --- | --- |
| 币对列表 | `BTCUSDT, ETHUSDT, SOLUSDT, BNBUSDT, XRPUSDT, DOGEUSDT, PENDLEUSDT, MONUSDT` | 去首尾空白、大写、移除非字母数字；最终长度 `3..30`；去重但保留顺序；至少保留一个 |
| 周期 | `1h` | `1m, 5m, 15m, 1h, 4h, 1d, 1w` |
| K 线数量 | `500` | 数值范围 `120..1000`；页面步长 `20` |
| Lag Start / `momStart` | `2` | 取整，最小 `2` |
| Lag End / `momEnd` | `52` | 取整，最小 `3`；必须严格大于 `momStart` |
| Z 窗口 / `zLength` | `52` | 取整，最小 `10` |
| 极端阈值 / `extThresh` | `2.0` | JavaScript 只要求有限数；控件建议范围 `0.5..5`，步长 `0.1` |
| 平滑 / `smoothLen` | `8` | 取整，最小 `1` |
| 熊市 WMA 周期 | `200` | 取整并夹在 `10..990`；实际 K 线数必须严格大于 WMA 周期 |
| 数据源 | `spot` | `spot, futures, bybitSpot, bybitLinear, bitgetSpot, bitgetFutures` |
| 自动刷新 | `0` | `0, 15, 30, 60, 300` 秒；`0` 表示关闭 |

当 K 线请求数与实返数一致且 `count <= WMA` 时提示“K线数量必须大于熊市 WMA 周期”；实返数较少时提示“行情仅返回 N 根 K 线，无法计算 WMA L”。

当前页面只持久化币对列表/顺序、数据源、自动刷新。Android 路线图已决定持久化所有图表设置；这是明确的 App 增强，不应被视为契约偏离。

## JT Regime 算法

以 K 线索引 `i` 计算：

```text
仅当 i >= zLength + momEnd：
  ret1[i]   = (close[i] - close[i-1]) / close[i-1]
  retMom[i] = (close[i-(momStart-1)] - close[i-momEnd]) / close[i-momEnd]

z1   = populationZScore(ret1, zLength, i)
zMom = populationZScore(retMom, zLength, i)
extremeUp   = z1 >  extThresh
extremeDown = z1 < -extThresh
shortTermFactor = (extremeUp || extremeDown) ? -z1 : z1
rawScore[i] = zMom + shortTermFactor
score[i] = WMA(rawScore, smoothLen, i)
```

必须保留的细节：

- SMA 窗口包含当前值；标准差使用总体方差，除以 `length` 而不是 `length - 1`。
- 窗口不足、含 `null`/非有限数，或标准差为零时，Z-score 返回 `0`。
- 因此预热期的 `rawScore` 是 `0`，指标从索引 `smoothLen - 1` 开始输出零值，而不是等到动量和 Z 窗口全部形成才输出。
- 极端判定是严格不等号；恰好等于阈值不算极端。极端时短期因子取反。
- WMA 权重从最新到最旧为 `length, length-1, ..., 1`，分母为 `length * (length + 1) / 2`。
- 不对中间值舍入、截断或字符串化；显示状态中的最新 Regime 才使用两位小数。

### 柱状图颜色

- `score >= 0` 且严格高于前一个已输出值：`rgba(0, 176, 80, 0.82)`。
- `score >= 0` 但未上升：`rgba(0, 176, 80, 0.42)`。
- `score < 0` 且严格低于前值：`rgba(255, 59, 48, 0.82)`。
- `score < 0` 但未下降：`rgba(255, 59, 48, 0.42)`。

## 熊市 WMA 叠加层

- 对 close 使用上述 WMA，第一个输出点在 K 线索引 `length - 1`。
- `close < WMA` 为熊市；相等不是熊市。
- WMA 是 2px 白线 `#ffffff`；熊市区域是 close 与 WMA 之间的 `rgba(0, 128, 0, 0.30)` 绿色填充。
- 相邻点在熊/非熊之间变化时，按 `close - WMA` 的线性零交叉分割填充，不把整段粗略涂绿。
- 叠加 canvas 不接收指针事件，并在尺寸、可见范围、指针、滚轮和双击改变后重绘。

## 抄底标记

遍历指标输出数组，从其索引 `2` 开始。常量不随用户设置变化：

```text
recentExtremeLookback = 12
breakoutLookback = 5
markerCooldown = 8
```

- 候选点必须在当前指标点及往前 12 个指标点中至少有一个 `isExtremeDown`；实际最多检查 13 个点，包含当前点。
- “确认抄底”：前值 `<= 0`、当前值 `> 0`，且当前 close 严格高于当前 K 线之前最多 5 根 K 线的最高 high。其优先级高于试探标记，命中后当轮 `continue`。
- “试探抄底”：当前值 `< 0`，前值 `<=` 前前值，且当前值严格大于前值；距上个任意类型标记的“指标输出数组索引”差必须 `>= 8`。
- 两类都在 K 线下方显示向上箭头。确认为 `#4e8cff`，试探为 `#ffcc00`；文字分别为“确认抄底”和“试探抄底”。
- 判定全部使用未舍入的浮点值。

## 图表展示与联动

- 深色背景 `#101216`；上涨 K 线 `#00b050`，下跌 K 线 `#ff3b30`；网格 `#20252d`。
- 价格 K 线在上、Regime 柱状图在下；手机竖屏和横屏都保持上下排列。振荡器价格轴上下边距均为 `0.12`。
- Regime 零线为 1px 虚线 `#7c8491`，显示轴标签和 `Zero` 标题；Regime 显示精度两位，`minMove=0.01`。
- 时间显示固定为 `Asia/Shanghai`/`zh-CN`；时间轴显示日期与分钟，不显示秒；完整时间包含年。
- 两张图的可见 logical range 双向同步：在任一张图缩放/平移，另一张跟随。同步过程需防止递归回调。
- 当前页面没有同步两图的十字光标；价格图十字光标只触发 WMA 叠加层重绘。Android 不得把“十字光标同步”误当成必须兼容行为。
- 每次成功换币对、改周期/数据源/刷新周期或点击重新加载后，两图都 `fitContent()`，即重置到整个数据范围。Android 旋转则按路线图决定保留视图状态，不触发重新加载和 `fitContent()`。
- 页面加载期间防止并发重入。自动刷新采用一次性 timer，在本次成功或失败后重新计时，不并发请求。Android 还需叠加“仅前台可见时刷新”的路线图决定。
- 成功状态显示 K 线数、最新 Regime（两位）、标记数、最近标记时间，以及最新点的“短期过热/短期恐慌”。

## 固定夹具 v1

回归测试应在 Android 项目中加入一份静态 JSON（建议路径 `android-app/app/src/test/resources/fixtures/jt-regime-v1.json`），而不在测试时请求真实交易所。该 JSON 由以下确定性规则一次性生成后必须作为静态文件提交：

```text
price = 100.0
for index in 0..79:
  if index < 30:      price += 0.1
  else if index < 40: price -= 0.1
  else if index == 40:price -= 10
  else if index < 43: price -= 2
  else if index < 48: price += 1
  else if index < 54: price += 4
  else:               price += 0.1
  close = round(price, 4)
  candle = {
    time: 1700000000 + index * 3600,
    open: close,
    high: close + 0.3,
    low: close - 0.3,
    close: close
  }
```

夹具设置固定为：

```json
{
  "interval": "1h",
  "limit": 200,
  "marketType": "spot",
  "momStart": 2,
  "momEnd": 10,
  "zLength": 10,
  "extThresh": 1.5,
  "smoothLen": 4
}
```

必须冻结的断言：

- 指标输出首点是 K 线索引 `3`，`time=1700010800`，`value=0`。
- 索引 `39`：`value=-1.265519052690`。
- 索引 `40`：`z1=-2.999999990384`，`isExtremeDown=true`，`value=0.065927102399`。
- 索引 `47`：`value=0.061237950808`。
- 标记按顺序为：
  1. `试探抄底`, `time=1700129600`, `score=-2.790338528417`, `close=102.3`
  2. `确认抄底`, `time=1700169200`, `score=0.061237950808`, `close=93.0`
  3. `试探抄底`, `time=1700226800`, `score=-1.409736600157`, `close=118.0`

再加一个 WMA 极小夹具：close 依次为 `[10, 11, 9, 8, 12]`，周期 `3`。WMA 从第三根开始，依次为 `59/6, 53/6, 61/6`，`isBear` 依次为 `[true, true, false]`。

### 浮点验收容差

- 算法中间值和最终 `value/z1/zMom/WMA`：使用 `abs(actual - expected) <= 1e-10` 或 `relative error <= 1e-10` 中较宽者。现有 Python/JavaScript 对照已在 10 位小数精度上一致。
- 布尔极端判定、标记数量/类型/顺序/时间/颜色和所属 K 线必须精确相等，不使用容差。
- 显示字符串另行断言两位小数，不用显示舍入值反推算法正确性。
