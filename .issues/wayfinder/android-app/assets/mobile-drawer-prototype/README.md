# JT Chart 移动端抽屉原型

> PROTOTYPE — 仅用于确认交互方向，不是正式 Android 实现，决策后应删除或吸收结论。

问题：在 Pixel 9 Pro XL 竖屏与横屏上，左侧 Material 3 设置抽屉应如何组织，才能兼顾图表优先、参数易用和手势不冲突？

三个结构不同的方案在同一页面中，通过 `?variant=A|B|C` 和页面底部浮动切换器选择。

## 运行

在 `jt-chart` 根目录执行：

```bash
python3 -m http.server 4174 --directory .issues/wayfinder/android-app/assets/mobile-drawer-prototype
```

打开 <http://127.0.0.1:4174/prototype.html?variant=A>。

## 评审操作

- 点左上角菜单打开抽屉，完成币种切换和参数修改。
- 点左右底部切换器，或在输入框外按 `←` / `→` 切换 A/B/C。
- 点右上角旋转按钮，模拟 Pixel 9 Pro XL 竖屏/横屏。
- 用顶部状态按钮切换正常、加载、离线缓存和错误状态。
- 抽屉关闭时在图表上横向拖动；抽屉打开时确认图表不再响应。

## 方案

- A — 即时生效分组：单层滚动抽屉，所有设置按组展开，变更即时刷新。
- B — 分页后应用：抽屉内部使用分类标签，先编辑草稿，再一次应用。
- C — 逐级聚焦：首层只显示摘要，点击某类后进入二级详情，为图表留出更多视觉空间。
