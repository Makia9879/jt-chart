## Question

后台策略引擎应如何复现当前 `calculateJTRegimeOscillator()` 和 `buildBottomMarkers()`，并通过确定性的同一输入对比证明“试探抄底”和“确认抄底”与页面一致？同时明确如何剔除未收盘 K 线、如何处理公式预热期及策略版本变化。

## Resolution

### 判定契约

- 策略输入采用按开盘时间升序的规范 Candle：`open_time_ms`、`close_time_ms`、`open`、`high`、`low`、`close`。在每次计算前，以 `now_utc_ms >= close_time_ms + 10_000` 为准过滤；最后一根未满足条件的 K 线绝不进入振荡器、标记或事件账本。
- `close_time_ms` 必须在行情适配层显式保留或推导，不能延续当前仅透传 `[open, high, low, close]` 的 API：Binance 使用原始行的 close time（第 7 个字段，闭合阈值为该值加 1 ms）；Bybit 与 Bitget 使用 `open_time_ms + interval_duration_ms` 推导。`1w` 按 UTC 连续 7 天处理。10 秒是已确认的数据落定宽限，调度器应在该时点之后扫描。
- 每一份监控配置冻结一个 `strategy_version`。第一版值为 `jt-regime-v1`，代表当前 HTML 的逐项语义；公式、常量、预热或信号条件的任何改变都必须升版本，不能覆盖旧版本事件的含义。

### 公式一致性

- 后台逐字复现 `calculateJTRegimeOscillator()`：`ret1` 与 `retMom` 只在 `i >= zLength + momEnd` 计算；总体样本标准差使用除数 `length`；极端值采用严格 `>` / `<`，极端时短周期因子取 `-z1`；WMA 最新值权重最大。
- 还必须复现 JavaScript 的预热细节：`zscore()` 在样本不完整时返回 `null`，随后 `zMom + shortTermFactor` 在 JS 数值运算中变为 `0`；因此 `rawScore` 在有效 Z-score 之前是 `0`，WMA 可自 `smoothLen - 1` 起输出。后台不得把这些早期分数改为缺失值，否则数组下标、冷却期与后续信号均可能不一致。
- `buildBottomMarkers()` 的常量固定为：最近极端下跌回看 12 个 oscillator 元素、结构突破回看 5 根原始 K 线、标记冷却 8 个 oscillator 元素。确认条件优先于试探条件；确认信号更新冷却索引并 `continue`；两类信号均作为独立事件类型。
- 默认参数下，第一个有效 Z-score 所在原始 K 线索引为 `2*zLength + momEnd - 1`（默认 155）；实际可发送信号还需要满足标记的历史条件。Worker 获取历史时必须至少覆盖该预热长度、WMA 长度和标记回看，且配置的 `limit` 不得小于该最小窗口；建议后台按 `max(configured_limit, 2*zLength + momEnd + smoothLen + 12 + 5 + 2)` 请求，以避免短窗口改变策略结果。

### 一致性测试方案

1. 将公式拆为无 I/O 的共享策略核心，并保留浏览器调用入口；Worker 使用同一核心或由同一规范生成的实现。测试不能只断言“有无信号”。
2. 固定一份按时间排序的原始 K 线夹具，覆盖：预热、极端下跌、试探、确认、8 根冷却、平盘（标准差为 0）、以及末尾未收盘 K 线。夹具同时保存期望的每根 `(candle_index, time, score, z1, zMom, isExtremeDown)` 与完整 marker 元组 `(time, text, score, close)`。
3. 对页面核心与 Worker 核心喂入完全相同的**已收盘**规范 Candle，逐字段比较；浮点数使用绝对误差 `<= 1e-12`，marker 的时间、类型、顺序必须完全相同。另加性质测试：向夹具尾部追加任意未收盘 K 线，所有可发送 marker 结果不得变化。
4. 适配器契约测试分别构造 Binance、Bybit、Bitget 原始行，验证升序、数值转换和 `close_time_ms`；再用边界时钟断言恰好在 close、close+9999 ms 均被排除，close+10000 ms 才被纳入。

### 已知兼容边界

当前页面会将交易所返回的最后一根（可能未收盘）直接参与绘图和标记；后台故意不与这一展示行为一致，而与已确认的正式通知语义一致。页面后续应标示最新未收盘 K 线，避免视觉标记被误解为已发送通知。
